Title: Automated Program Repair Based on REST API Specifications Using Large Language Models
Authors: Katsuki Yamagishi, Norihiro Yoshida, Erina Makihara, Katsuro Inoue
ID: 2510.25148
URL: https://arxiv.org/abs/2510.25148

Abstract:
Many cloud services provide REST API accessible to client applications. However,
developers often identify specification violations only during testing, as error
messages typically lack the detail necessary for effective diagnosis.
Consequently, debugging requires trial and error. This study proposes dcFix, a
method for detecting and automatically repairing REST API misuses in client
programs. In particular, dcFix identifies non-conforming code fragments,
integrates them with the relevant API specifications into prompts, and leverages
a Large Language Model (LLM) to produce the corrected code. Our evaluation
demonstrates that dcFix accurately detects misuse and outperforms the baseline
approach, in which prompts to the LLM omit any indication of code fragments non
conforming to REST API specifications.
