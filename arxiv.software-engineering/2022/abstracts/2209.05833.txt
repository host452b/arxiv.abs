Title: White-Box and Black-Box Fuzzing for GraphQL APIs
Authors: Asma Belhadi, Man Zhang, Andrea Arcuri
ID: 2209.05833
URL: https://arxiv.org/abs/2209.05833

Abstract:
The Graph Query Language (GraphQL) is a powerful language for APIs manipulation
in web services. It has been recently introduced as an alternative solution for
addressing the limitations of RESTful APIs. This paper introduces an automated
solution for GraphQL APIs testing. We present a full framework for automated
APIs testing, from the schema extraction to test case generation. In addition,
we consider two kinds of testing: white-box and black-box testing. The white-box
testing is performed when the source code of the GraphQL API is available. Our
approach is based on evolutionary search. Test cases are evolved to
intelligently explore the solution space while maximizing code coverage and
fault-finding criteria. The black-box testing does not require access to the
source code of the GraphQL API. It is therefore of more general applicability,
albeit it has worse performance. In this context, we use a random search to
generate GraphQL data. The proposed framework is implemented and integrated into
the open-source EvoMaster tool. With enabled white-box heuristics, i.e.,
white-box mode, experiments on 7 open-source GraphQL APIs show statistically
significant improvement of the evolutionary approach compared to the baseline
random search. In addition, experiments on 31 online GraphQL APIs reveal the
ability of the black-box mode to detect real faults.
