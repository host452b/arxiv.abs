Title: Vulnerabilities in AI Code Generators: Exploring Targeted Data Poisoning
Attacks
Authors: Domenico Cotroneo, Cristina Improta, Pietro Liguori, Roberto Natella
ID: 2308.04451
URL: https://arxiv.org/abs/2308.04451

Abstract:
AI-based code generators have become pivotal in assisting developers in writing
software starting from natural language (NL). However, they are trained on large
amounts of data, often collected from unsanitized online sources (e.g., GitHub,
HuggingFace). As a consequence, AI models become an easy target for data
poisoning, i.e., an attack that injects malicious samples into the training data
to generate vulnerable code.
  To address this threat, this work investigates the security of AI code
generators by devising a targeted data poisoning strategy. We poison the
training data by injecting increasing amounts of code containing security
vulnerabilities and assess the attack's success on different state-of-the-art
models for code generation. Our study shows that AI code generators are
vulnerable to even a small amount of poison. Notably, the attack success
strongly depends on the model architecture and poisoning rate, whereas it is not
influenced by the type of vulnerabilities. Moreover, since the attack does not
impact the correctness of code generated by pre-trained models, it is hard to
detect. Lastly, our work offers practical insights into understanding and
potentially mitigating this threat.
