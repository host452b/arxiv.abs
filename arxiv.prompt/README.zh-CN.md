## 各章节 80/20 最佳实践

> **二八定律：** 每个提示词工程领域中，20% 的关键技巧决定 80% 的可测量 LLM 性能提升。下列关键杠杆均来自 4,780 篇提示词工程论文的高频共识。

### I. 推理与思维链（#1–6）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **思维链（CoT）**：添加"逐步思考"或演示推理轨迹 | 文献中引用最多的提示技巧；在数学、逻辑和多步问答上持续提升，零训练成本 |
| **ReAct**：交织推理轨迹与行动步骤（思考 → 行动 → 观察） | 将 CoT 扩展到 Agent 场景；ALFWorld +34% / WebShop +10% vs 纯 RL；覆盖 80% 的推理-行动用例 |

> CoT 处理推理质量，ReAct 将其扩展到行动——两者共同覆盖 80% 的推理提示工程收益。

---

### II. 示例与上下文学习（#7–11）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **多样化示例选择**：选择语义多样且代表任务分布的演示样本 | 示例质量（而非数量）决定 ICL 性能；3 个精选示例通常优于 10 个随机示例 |
| **自一致性**：采样多条推理路径，取多数投票 | 在推理任务上比单路径 CoT 提升 5–15%；无需额外训练，仅需推理计算 |

> 示例质量 + 自一致性共同覆盖 80% 的 ICL 性能优化。

### III. 设计与格式化（#12–16）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **角色设定**（"你是一位专业的 X..."）：分配人格和知识领域 | 在领域特定任务上持续提升；是任何提示词中最高 ROI 的单行添加之一 |
| **结构化输出格式**（JSON / XML / Markdown Schema）：指定精确的输出结构 | 消除后处理失败；是程序化 LLM 使用中最有影响的单一格式选择 |

> 角色设定聚焦 LLM，结构化输出使其可用——两者共同覆盖 80% 的提示设计质量。

### IV. 编排与工作流（#17–22）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **提示链**：将复杂任务分解为顺序提示，每个建立在前一个输出之上 | 避免单提示上下文过载；覆盖 80% 的多步任务编排需求 |
| **多 Agent 委托**：将子任务路由给有边界领域的专业 Agent | 降低幻觉，提高具有不同技能需求任务的质量；MetaGPT / AutoGen 范式 |

> 链式处理顺序复杂性，多 Agent 处理技能多样性——两者共同覆盖 80% 的复杂 LLM 工作流编排。

### V. 优化与压缩（#23–28）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **自动提示优化（APO / OPRO）**：使用 LLM 反馈迭代改进提示措辞 | 在大多数基准上优于人工编写的提示；"编译你的提示"范式 |
| **提示压缩**：删除冗余 token 同时保留语义内容（LLMLingua 风格） | 3–20× token 减少，性能损失 < 5%；直接降低部署系统的成本和延迟 |

> APO 最大化质量，压缩最大化效率——两者共同覆盖 80% 的提示优化工程。

### VI. 质量与鲁棒性（#29–33）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|
| **LLM 作为评判**：使用强力 LLM（GPT-4 / Claude）按照评分标准评估输出 | 以 1/100 的成本与人类判断的相关性达 80%+；标准可扩展评估方法 |

> LLM 评判衡量质量，敏感性测试衡量可靠性——两者共同覆盖 80% 的提示质量保证需求。

### VII. 防御与保护（#34–41）

| 关键杠杆（20%） | 覆盖效果（80%） |
|----------------|----------------|

| **输入净化 + 白名单验证**：过滤或转义危险模式；只允许预期的输入模式 | 在到达 LLM 之前在输入边界阻断 80% 的提示注入攻击 |

> 聚光灯保护指令完整性，输入净化在边界处防御——两者共同覆盖 80% 的提示注入防御。

| 关键杠杆（20%） | 覆盖效果（80%） |

> 人格越狱是主要的直接攻击，间接注入是主要的 Agent 攻击——理解两者覆盖 80% 的 LLM 攻击面。

## Paradigm Shifts / 范式转移 (Kuhn)

| 旧范式 | 触发事件 | 新范式 |
|---|---|---|
| **基于规则的 NLP** — 手工编码模板和语法 | GPT-3 / 少样本学习 (2020): 上下文内学习 | **基于提示词的泛化** — 新任务无需微调 |
| **按任务微调** — 每个任务需要独立标注数据集 | InstructGPT / RLHF (2022): 对齐训练 | **指令跟随基础模型** — 一个模型通过提示处理任意任务 |
| **人工编写固定提示词** — 手工艺式提示工程 | APO / OPRO (2023): LLM 评估 LLM 优化 | **自动化提示优化** — 将提示词编写视为编译步骤 |
| **静态提示词模板** — 固定的上下文窗口 | RAG + 工具调用 (2023+): 外部知识访问 | **动态检索增强提示词** — 有依据、始终最新的上下文 |

**已被推翻的认知误区:**
- ✗ "更长的提示词 = 更好的结果" → 质量 > 数量
- ✗ "同一个提示适用于所有模型" → 不同模型需要模型专属优化
- ✗ "CoT总能提升性能" → 简单任务上CoT引入冗余推理

## 被推翻的主流认知 (Superseded Theories)

以下曾在提示词工程领域被普遍接受的认知，已被本集合4,780篇论文的研究成果所推翻：

| 章节 | 被推翻的理论 / 既定谬误 |
|---|---|
| 推理与思维链 | "更长的提示词总是产生更好的推理" → 质量和明确性胜过长度；CoT在简单任务上增加噪声 |
| 示例与上下文学习 | "更多少样本示例 = 更好的ICL性能" → 3个精心挑选的示例胜过10个随机示例；多样性胜过数量 |
| 设计与格式 | "同一个提示适用于所有LLM" → 不同模型架构需要模型专属提示优化 |
| 编排与工作流 | "单个提示可以处理复杂的多步任务" → 通过提示链进行任务分解是复杂工作流的必要条件 |
| 优化与压缩 | "人工编写的提示是最优的" → APO/OPRO（LLM评估LLM）在大多数基准测试中持续优于人工提示 |
| 质量与鲁棒性 | "一致的输出意味着可靠的提示" → 敏感性测试揭示在改述变体上会失败的脆弱提示 |
| 防御与保护 | "系统提示提供绝对安全" → 越狱和提示注入可靠地绕过系统级指令 |
| 攻击与安全研究 | "安全对齐的模型能抵抗所有对抗性提示" → 自动越狱（GCG、PAIR）可靠地从对齐模型中引出违规行为 |
