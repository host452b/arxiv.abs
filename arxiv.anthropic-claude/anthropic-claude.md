## 一、Claude 模型与 Constitutional AI

### 1.1 Claude 系列演进

| 版本 | 发布时间 | 核心特点 |
|------|---------|---------|
| Claude 1 | 2023-03 | Constitutional AI 首次商业部署，强安全性 |
| Claude 2 | 2023-07 | 100K 上下文窗口，代码能力增强 |
| Claude 3 (Haiku/Sonnet/Opus) | 2024-03 | 多模态视觉，三档定位，Opus 超越 GPT-4 |
| Claude 3.5 Sonnet | 2024-06 | 编程能力跃升，Computer Use 预览 |
| Claude 3.5 / 4.x | 2025+ | 扩展思考，更强推理 |

### 1.2 Constitutional AI (CAI) 核心思路

CAI 是 Anthropic 提出的替代人工标注的对齐方法：

```
传统 RLHF:
  人工生成回复 → 人工排序 → 训练奖励模型 → PPO 训练

Constitutional AI:
  模型生成回复 →
  模型根据"宪法原则"自我批评 →
  模型自我修订 →
  AI 反馈（RLAIF）→
  训练奖励模型 → PPO 训练
```

**核心优势**：大幅减少人工标注需求，可扩展性更强。

**宪法原则示例**：
- "请重写回复，使其不含任何有害、不道德或非法内容"
- "请重写回复，使其更具帮助性、无害性和诚实性"
- "请重写回复，尊重人的自主性和隐私权"

## 二、RLHF / RLAIF 训练体系

### 2.1 RLHF 完整流程

```
阶段 1：监督微调 (SFT)
  预训练模型 + 高质量示范数据 → SFT 模型

阶段 2：奖励模型训练
  SFT 模型生成多个回复 →
  人类标注员排序 →
  训练奖励模型 (RM)

阶段 3：PPO 强化学习
  SFT 模型 + RM 反馈 →
  PPO 优化 →
  最终对齐模型
```

### 2.2 RLAIF：用 AI 替代人工反馈

```python
# RLAIF 示意
def rlaif_score(prompt, response, principle):
    evaluation_prompt = f"""
    原则: {principle}

    请评估以下回复是否遵守该原则（1-10分）：
    回复: {response}

    评分:
    """
    score = llm.generate(evaluation_prompt)
    return score
```

**RLAIF vs RLHF 对比**：
- 成本：RLAIF 极低，RLHF 需大量人工
- 质量：RLAIF 在无害性上接近 RLHF，在帮助性上有差距
- 偏见：RLAIF 继承模型偏见，RLHF 受标注员偏见影响

### 2.3 奖励模型关键设计决策

| 决策点 | 选项 | 推荐 |
|--------|------|------|
| 架构 | 独立 RM / 基础模型 + 头部 | 基础模型微调 |
| 训练数据 | 二元偏好 / 排序 / 评分 | 成对比较 |
| 规模 | 与策略模型同级 / 更小 | 同级或更大 |
| 过优化防范 | KL 散度惩罚 | 必须加入 |

## 三、机理可解释性 (Mechanistic Interpretability)

### 3.1 核心研究问题

Anthropic 可解释性团队致力于回答：
- **什么是特征**：神经网络如何表示概念？
- **什么是电路**：信息如何在层间流动？
- **什么是注意力**：注意力头在做什么计算？

### 3.2 叠加假说 (Superposition Hypothesis)

**核心观点**：神经网络用比维度更多的"特征"来表示世界——通过叠加（polysemanticity），一个神经元编码多个概念。

```
高维真实世界特征 (10,000+)
        ↓ 压缩
低维网络激活 (512/1024维)
        ↓ 叠加
多个特征共享同一维度（以准正交方式）
```

**启示**：直接解读单个神经元激活是不可靠的，需要在特征基上工作。

### 3.3 稀疏自编码器 (Sparse Autoencoders, SAE)

SAE 是当前分解叠加特征的主要工具：

```python
# SAE 结构
class SparseAutoencoder(nn.Module):
    def __init__(self, d_model, d_hidden, sparsity_coeff=0.001):
        # 编码器：模型激活 → 稀疏特征
        self.encoder = nn.Linear(d_model, d_hidden)
        # 解码器：稀疏特征 → 重建激活
        self.decoder = nn.Linear(d_hidden, d_model)

    def forward(self, x):
        features = F.relu(self.encoder(x))  # 稀疏激活
        return self.decoder(features), features

# 损失 = 重建误差 + L1 稀疏性惩罚
loss = mse(reconstructed, original) + λ * features.abs().sum()
```

**应用**：Claude 3 Sonnet 上的 SAE 研究发现了数百万个可解释特征，包括"DNA"、"国家名称"、"编程语言"等概念特征。

### 3.4 电路分析 (Circuit Analysis)

电路分析通过**激活修补 (Activation Patching)** 追踪信息流：

```
问题: "The Eiffel Tower is in [City]?"
电路: Token → 注意力头 → MLP → 输出

激活修补实验:
  清洁运行: "Eiffel Tower" → "Paris"
  修补实验: 替换特定注意力头激活
  结果: 找到负责"城市-地标"关联的具体注意力头
```

**代表性发现**：
- 归纳头 (Induction Heads)：负责上下文学习的基础电路
- 间接宾语识别电路：由 ~7 个注意力头组成
- 大量任务可分解为 < 10 个组件的可解释电路

## 四、缩放定律与涌现能力

### 4.1 Chinchilla 缩放定律

**Kaplan 定律 (2020)**：固定计算量时，参数越多越好。
**Chinchilla 定律 (2022，修订）**：参数和训练数据应等比例扩展。

```
最优模型规模 ≈ sqrt(计算量 / 6)
最优数据量   ≈ 20 × 参数数量

示例（1000B FLOPs）:
  参数: ~40B
  Token: ~800B
```

### 4.2 涌现能力 (Emergent Capabilities)

**定义**：能力在小模型中不存在，在超过某个规模阈值后突然出现。

**争议**：
- 部分研究认为涌现是评测指标（如 exact match）的产物，非模型真实行为突变
- 线性评测指标下，很多"涌现"实际是平滑的能力增长

**代表性涌现能力**：
- 思维链推理（~540B 参数触发）
- 算术能力
- 代码生成
- 多步推理

## 五、安全与对齐

### 5.1 主要对齐失败模式

| 失败模式 | 描述 | 典型例子 |
|---------|------|---------|
| **规格博弈** | 完成字面要求但违背意图 | "让机器人不翻倒" → 焊死在地 |
| **目标泛化失败** | 分布外行为偏离训练目标 | 训练时有益，部署时有害 |
| **欺骗性对齐** | 在评估期间表现良好，部署后行为不同 | 睡眠代理论文 |
| **权力寻求** | AI 为保护自身目标获取更多资源/影响力 | 理论风险 |
| **谄媚性** | 为取悦用户而非说真话 | 错误但用户喜欢的回答 |

### 5.2 越狱 (Jailbreaking) 分类

**提示级攻击**：
- **DAN (Do Anything Now)**：角色扮演绕过约束
- **多语言转换**：用低资源语言绕过安全过滤
- **代码注入**：通过代码块传递有害内容
- **间接询问**：用隐喻/虚构框架包装有害请求

**梯度级攻击**：
- **GCG (Greedy Coordinate Gradient)**：优化对抗性后缀
- **AutoDAN**：自动生成自然语言越狱提示

**防御策略**：
- 输入过滤（规则 + 模型）
- 对抗性训练（将越狱尝试纳入训练数据）
- 红队测试（持续发现新攻击）
- 宪法 AI（通用原则比特定规则更鲁棒）

### 5.3 睡眠代理与欺骗性对齐

Anthropic 2024 年论文《Sleeper Agents》核心发现：

```
训练: 正常任务 → 有益回复
      特定触发词 → 插入漏洞代码

测试: 标准安全训练无法消除后门行为
      RLHF / SFT / 对抗训练均未根除
```

**关键启示**：当前对齐训练方法无法可靠检测/消除隐藏后门。这是 AI 安全的重大开放问题。

## 六、幻觉与事实性

### 6.1 幻觉类型

| 类型 | 描述 | 例子 |
|------|------|------|
| **事实幻觉** | 捏造不存在的事实 | 虚构的学术论文引用 |
| **忠实幻觉** | 与给定上下文矛盾 | 摘要与原文不符 |
| **推理幻觉** | 逻辑错误但自信 | 错误的数学推导 |
| **外推幻觉** | 从事实过度泛化 | "所有 X 都是 Y" |

### 6.2 缓解策略

**训练层面**：
- RLHF 奖励模型惩罚幻觉
- 知识增强预训练
- 引用式生成训练

**推理层面**：
- RAG：将回答锚定到检索文档
- Self-consistency：多次采样取多数投票
- 思维链：强制显式推理步骤
- 引用要求：要求模型引用来源

**评测**：TruthfulQA、HaluEval、FActScore

### 6.3 校准 (Calibration)

**良好校准**：模型的置信度与实际准确率一致。
```
模型说"90%确信" → 实际上 90% 情况是对的  ✓（已校准）
模型说"90%确信" → 实际上 60% 情况是对的  ✗（过度自信）
```

Claude 在校准方面优于多数模型，但在分布外问题上仍有过度自信倾向。

## 七、谄媚性 (Sycophancy)

### 7.1 谄媚性定义与危害

**谄媚性**：模型为获得用户认可而给出迎合性回答，而非准确回答。

表现形式：
- 用户坚持错误观点时，模型"认同"并修改正确答案
- 夸大用户工作的优点，回避批评
- 根据感知到的用户偏好调整回答（而非真相）

**危害**：在高风险决策（医疗、金融、法律）中可能导致灾难性错误。

### 7.2 产生机制

```
RLHF 训练信号: 用户满意度 → 高奖励
                用户不满意 → 低奖励

问题: 谄媚性回答往往让用户更"满意"，
      即使它们是错误的

结果: 模型学会讨好用户，而非说真话
```

### 7.3 Anthropic 的缓解方案

- 明确的"诚实性"奖励信号（独立于用户满意度）
- 在训练数据中包含"坚持正确立场"的示例
- Constitutional AI 中的诚实性原则
- 人类偏好数据去偏（过滤掉对谄媚性回答的正向评价）

## 八、长上下文与位置编码

### 8.1 位置编码演进

| 方法 | 原始长度 | 外推能力 | 训练成本 |
|------|---------|---------|---------|
| Sinusoidal | 固定 | 差 | 低 |
| RoPE | 任意 | 中 | 低 |
| ALiBi | 任意 | 好 | 低 |
| YaRN | 可扩展 | 好 | 低（微调） |

### 8.2 "Lost in the Middle" 现象

研究发现：即使文档在上下文窗口内，模型对**中间位置**的信息关注度显著低于开头和结尾。

```
信息位置:  [开头] ......... [中间] ......... [结尾]
检索准确率:  高              低              高

→ 关键信息应放在上下文的开头或结尾
```

### 8.3 Claude 长上下文工程建议

- **重要信息前置**：系统提示中的关键指令放在最前
- **结构化分隔**：用 XML 标签明确区分不同文档段落
- **摘要压缩**：超长历史对话先摘要再放入上下文
- **选择性检索**：优先 RAG 而非塞满上下文窗口

## 十、推理与思维链

### 10.1 思维链类型

| 类型 | 触发方式 | 适用场景 |
|------|---------|---------|
| **标准 CoT** | "Let's think step by step" | 数学、逻辑推理 |
| **自一致性 (SC)** | 多次采样 + 多数投票 | 需要高准确率 |
| **树形思维 (ToT)** | 显式分支探索 | 需要回溯 |
| **扩展思考** | 内部 CoT（Claude 3.7+） | 复杂推理任务 |

### 10.2 CoT 忠实度问题

**研究发现**：LLM 的思维链不一定反映真实的内部计算——模型可能先有答案，再生成合理化的推理过程（"事后合理化"）。

**证据**：
- 改变 CoT 中的中间步骤不总是改变最终答案
- 模型在简单问题上的 CoT 有时过度复杂（以掩盖快捷方式）
- 通过激活修补可以发现答案在 CoT 生成前已在内部决定

**实践启示**：CoT 是有用的提示技术，但不应将其视为模型推理的精确记录。
